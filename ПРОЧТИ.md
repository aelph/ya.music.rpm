# Yandex Music RPM Workaround for Fedora/openSUSE

Этот репозиторий содержит инструмент для адаптации официального `.deb` пакета Яндекс Музыки для использования в RPM-дистрибутивах (Fedora, openSUSE).

## Особенности сборки

- **Hi-Res Audio:** Подтверждена стабильная работа на частотах до **176,4 КГц** через ALSA.
- **Bluetooth:** Оптимизировано для работы с кодеком LDAC (проверено на Sony WH-XB900N, 88.2 КГц).
- **Название пакета** Alien автоматически инкрементирует версию пакета для корректной работы пакетного менеджера. Вы получите надпись Beta, там где в приложении указывается номер сборки.
- **Интеграция:** Исправлены категории меню (теперь отображается в разделе "Музыка") и исправлено отображение иконок в панели задач (StartupWMClass). Эти изменения только в ярлыке приложения. Сам исходный код не изменялся.

## Как использовать

1. Склонируйте репозиторий.
2. Установите зависимости: `sudo dnf install alien curl desktop-file-utils`.
3. Соберите RPM обычным пользователем: `./build_rpm.sh`.
4. Установите получившийся RPM штатными средствами вашей системы или используйте `./install_rpm.sh`, если нужен управляемый сценарий установки и проверки.

`build_rpm.sh` это простой сборщик. Он только скачивает `.deb`, правит ярлык в дереве сборки Alien и собирает RPM. Для него `sudo` не нужен. Результат сборки можно установить в систему штатными средствами работы с пакетами.

`install_rpm.sh` нужен только если вы хотите контролировать процесс установки или отладки. Он отвечает за привилегированную часть: ищет готовый RPM в директории проекта, при необходимости вызывает `./build_rpm.sh`, устанавливает пакет, обновляет базу desktop-файлов и проверяет результат.

## Почему не готовый RPM?

Чтобы не нарушать лицензионные права правообладателя, в этом репозитории бинарный файл приведён только в качестве примера успешной работы скрипта.

Скрипты только автоматизируют процесс конвертации официального пакета, предоставленного Яндексом, для вашей локальной системы. 

Никаких изменений в исходный код приложения не вносится, производится только пересборка пакета с использованием инструмента alien, включая правку ярлыка внутри сгенерированного дерева сборки RPM перед финальной сборкой.

## Обновление

Не запускайте обновление приложения изнутри самой программы. Как только Яндекс Музыка покажет уведомление о новой версии, закройте приложение.

После этого:

1. Либо выполните `./build_rpm.sh`, а затем установите собранный RPM штатными средствами системы.
2. Либо запустите `./install_rpm.sh`, и он сам при необходимости соберёт RPM и установит его в систему.
