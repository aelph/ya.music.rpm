# Yandex Music RPM Workaround for Fedora/openSUSE

Этот репозиторий содержит инструмент для адаптации официального `.deb` пакета Яндекс Музыки для использования в RPM-дистрибутивах (Fedora, openSUSE).

## Особенности сборки

- **Hi-Res Audio:** Подтверждена стабильная работа на частотах до **176,4 КГц** через ALSA.
- **Bluetooth:** Оптимизировано для работы с кодеком LDAC (проверено на Sony WH-XB900N, 88.2 КГц).
- **Название пакета** Alien автоматически инкрементирует версию пакета для корректной работы пакетного менеджера. Вы получите надпись Beta, там где в приложении указывается номер сборки.
- **Интеграция:** Исправлены категории меню (теперь отображается в разделе "Музыка") и исправлено отображение иконок в панели задач (StartupWMClass). Эти изменения только в ярлыке приложения. Сам исходный код не изменялся.
- **Обновления:** В ярлык приложения добавлен пункт контекстного меню «Обновить Яндекс Музыку (RPM)», а скрипты обновления запекаются в сам RPM-пакет. Поддерживаются оба режима запуска: из командной строки и графический — с окном запроса прав, индикатором выполнения и финальным уведомлением о результате.

## Обновление приложения

Не используйте встроенный в приложение механизм обновления: он рассчитан на `.deb` и в RPM-окружении только портит установку. Если приложение предложило обновиться — откажитесь.

![Рисунок 1](./attachments/pic01.png)

Если при закрытии приложения появится запрос пароля суперпользователя — тоже откажитесь.

![Рисунок 2](./attachments/pic02.png)

Вместо этого воспользуйтесь дополнительным пунктом контекстного меню ярлыка: «Обновить Яндекс Музыку (RPM)». Он доступен из меню приложений вашей оболочки рабочего стола (GNOME Activities, KDE Application Launcher и т.п.) по правому клику на иконке «Яндекс Музыка».

![Рисунок 3](./attachments/pic03.png)

Введите пароль в стандартном системном окне запроса привилегий и дождитесь окна с результатом обновления.

![Рисунок 4](./attachments/pic04.png)

Доступные способы запуска обновления:

1. Пункт контекстного меню ярлыка «Обновить Яндекс Музыку (RPM)» — графический сценарий с индикатором прогресса и окном о результате (требуется `zenity` или `kdialog`).
2. Команда `yandexmusic-rpm-update` в терминале — тот же сценарий; при наличии `zenity`/`kdialog` будет использован графический режим, иначе — текстовый.
3. `./install_rpm.sh` вручную из этого репозитория — для отладки или ручного контроля.
4. `./build_rpm.sh` с последующей установкой полученного RPM штатными средствами системы — для случаев, когда нужна только сборка без установки.

`yandexmusic-rpm-update` представляет собой тонкую обёртку над `/opt/Яндекс Музыка/updater/gui_wrapper.sh`, которая определяет доступный GUI-тулкит и запрашивает повышение полномочий через `pkexec`. Далее вызывается `/opt/Яндекс Музыка/updater/install_rpm.sh`: он пересобирает RPM из актуального `.deb` во временной директории и устанавливает полученный пакет.

## Как использовать

1. Склонируйте репозиторий.
2. Установите зависимости: `sudo dnf install alien curl desktop-file-utils`. Дополнительно для графического режима обновления рекомендуется установить `zenity` (GTK) либо `kdialog` (KDE).
3. Соберите RPM обычным пользователем: `./build_rpm.sh`.
4. Установите получившийся RPM штатными средствами вашей системы или используйте `./install_rpm.sh`, если нужен управляемый сценарий установки и проверки.

`build_rpm.sh` это простой сборщик. Он только скачивает `.deb`, правит ярлык в дереве сборки Alien и собирает RPM. Для него `sudo` не нужен. Результат сборки можно установить в систему штатными средствами работы с пакетами.

`install_rpm.sh` нужен только если вы хотите контролировать процесс установки или отладки. Он отвечает за привилегированную часть: ищет готовый RPM в директории проекта, при необходимости вызывает `./build_rpm.sh`, устанавливает пакет, обновляет базу desktop-файлов и проверяет результат.

## Почему не готовый RPM?

Чтобы не нарушать лицензионные права правообладателя, в этом репозитории бинарный файл приведён только в качестве примера успешной работы скрипта.

Скрипты только автоматизируют процесс конвертации официального пакета, предоставленного Яндексом, для вашей локальной системы. 

Никаких изменений в исходный код приложения не вносится, производится только пересборка пакета с использованием инструмента alien, включая правку ярлыка внутри сгенерированного дерева сборки RPM перед финальной сборкой.
